# App Links Artifacts

These files are rendered for the production hosts:

- `https://dineinmt.ikanisa.com/.well-known/assetlinks.json`
- `https://dineinmt.ikanisa.com/.well-known/apple-app-site-association`
- `https://dineinrw.ikanisa.com/.well-known/assetlinks.json`
- `https://dineinrw.ikanisa.com/.well-known/apple-app-site-association`

Render Malta artifacts with:

```bash
PLAY_APP_SIGNING_SHA256_MT="AA:BB:..." \
APPLE_TEAM_ID_MT="ABCDE12345" \
./scripts/render_app_links.sh --flavor mt --output-dir /tmp/dinein-mt-app-links
```

Render Rwanda artifacts with:

```bash
PLAY_APP_SIGNING_SHA256_RW="AA:BB:..." \
APPLE_TEAM_ID_RW="ABCDE12345" \
./scripts/render_app_links.sh --flavor rw --output-dir /tmp/dinein-rw-app-links
```

Inputs:

1. `PLAY_APP_SIGNING_SHA256_MT` or `PLAY_APP_SIGNING_SHA256_RW`: the Play App Signing SHA-256 fingerprint for the selected flavor package ID.
2. `APPLE_TEAM_ID_MT` or `APPLE_TEAM_ID_RW`: the Apple Developer Team ID that signs the selected flavor bundle ID.

`build_web_release.sh` and `validate_release_integrations.sh` will also render
the `.well-known` files automatically when those environment variables are set.
Publish the rendered files from the chosen output directory or from the build
artifact generated by the release script.

Serve `apple-app-site-association` with `application/json` content type and no
file extension.

The mobile apps are configured to handle:

- `https://dineinmt.ikanisa.com/v/{slug}`
- `https://dineinrw.ikanisa.com/v/{slug}`
